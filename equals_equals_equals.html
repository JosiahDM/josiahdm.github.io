Java Basics: equals, equals, EQUALS, Oh My!

Or, the difference between "=", "==", and ".equals()"

=

The fancy term for the "=" sign is the assignment operator. It assigns values to variables. So if you write int x = 10; the variable x now contains the integer 10. It will contain 10 until another assignment operator is used to change it's value.

You can do math after the "=" sign. So if you write x = 5 + 5; the program will run the calculation then assign the value 10 to x.

You can even run functions after the "=" sign. For example: int x = calculateSomething(); Will run the calculateSomething() function and whatever it returns will be stored in x.

Also it's nice to know there are various shortcuts you can use. A common thing we need to do is add some value to a variable. One way we can do it is say x = x + 5; What this does is takes whatever is in x, and adds 5. So if x started at 10 here, x will end up being 15.

This works, but we have a nice shortcut for it: the += operator. Using this operator, we would just write x += 5; This does exactly the same thing—adds 5 to whatever value is in x—but is a bit less typing. You can get the same functionality with any of the basic arithmetic operations: -=, *=, /=, and %=.

There are 2 more operators that kind of relate to the assignment operator, and those are the increment and decrement operators. So if you want to add or subtract one from a variable, it's simply ++ or -- following the variable. If we had x = 10; Then we do x++ and now the value of x is 11. Decrement works in the same way, subtracting 1: x--; These are useful in loop counters. (There is also ++x and --x which increment immediately before returning the value. But that can be saved for another post.)

==

== is known as the "equal to" operator. It's part of a group of operators called boolean operators, which are used to compare values (or object references). The == operator determines whether one value is equal to another value. So if we were to write (10 == 10) that would evaluate to true. If we previously set a variable x to 10, then we could write (x == 10) and that would evaluate to true. We can even evaluate variables such as (x == y) which would be either true or false depending on the values stored in each variable.

== is generally used in if/else statements and loops to check for conditions. For example: 

if (x == 1) {
  // Do this
}
else {
  // Do this
}

.equals()

The thing about == is it's very literal. It only becomes true if the things being evaluated really are equal to each other. This works very intuitively for primitive types like int and double, but doesn't work if trying to compare strings. This is where .equals() comes in to play.

Lets say we have two string variables named a, and b. To initialize them we might do String a = "abcd"; String b = "abcd"; . This actually creates two Objects of type "String". So if we do (a == b) it will be false, because a and b are different object references. Now, if we did String a = "abcd"; String b = a; then they would refer to the same object since we set b = a. In this case (a == b) would be true.

Therefore, to compare Strings for equality, we have to use the String method .equals() . This method checks each character within the Strings to be compared. If the characters are all the same, then it will return true. If not, false.

For our previous example, using .equals() with a and b, we could write: a.equals(b); which returns true, or b.equals(a); for the same result.

To recap:

= Assignment operator. Assigns values to variables.

== Boolean operator. Compares two values, returns true if they are equal, false if not.

.equals() A method of the String class. Compare two strings, if their values are equal return true, false if not.
