Switch and If/else are two types of program control statements. They allow you to control the flow of code execution based on the conditions in the statements. This is one of the fundamental pieces of any programming language, because we need to be able to say "if this is true, do this." Without this ability, our programs would be pretty inflexible and much less useful.

How they work:

First I'll talk about if/else. It's pretty much just like it sounds. "If one condition, do this. Else (otherwise), do this." Here is the syntax in Java:

if(condition) {
    //statement 1;
}
else {
    //statement 2;
}

With this if/else we get two branches of execution. If the condition within parentheses is true, statement 1 will execute, and statement 2 will not run. If the condition within parentheses is false, statement 1 will not run, and statement 2 will be executed. In either case, the program flow continues directly after the final closing brace.

If/else can also be expanded with "else if..." additions and even nested if statements. I won't go into that here since my focus is to compare the if/else to switch statements.

With that said, here is the form of a switch statement:

switch(expression) {
case constant1:
    statement 1;
    break;
case constant2:
    statement 2;
    break;
case constant3:
    statement 3;
    break;
default:
    default statement;
}

Within the parentheses is an expression. If that expression is equal to one of the constants, that branch of the switch statement is executed. So for instance if expression was equal to constant2, statement2 will execute. Followed by the break; statement, which will break the flow out of the switch statement, and continue after the closing brace. The break is important because without it the program would continue to the next case statement.

So what's the difference? Why use one over the other?

The difference is that if/else can select based on broader, more expressive conditions. You can use tests like less than, greater than, etc. This is useful if you don't know the specific values that would be needed to trigger a switch statement, or maybe the potential values are within a large range.

The switch statement requires literal values of type byte, short, int, char, enum, or String (in JDK7 and later). This means for a switch statement to make sense, you would either need a function that returns a value of these types, or a direct input of these types that matches the values within the switch statement. If/else statements only need conditions which can be evaluated to true or false. This makes if/else statements a bit more flexible and possibly easier to implement for most purposes.

One thing a switch statement can be more useful for is user input menus. Since these menus will have a known, relatively small set of specific options, switch statements are probably more efficient to use in this case. Switch could also be useful for error or status codes. If you have some program running that returns codes based on success or failure, you can use switch statements to control what happens next given that certain known, static code.
